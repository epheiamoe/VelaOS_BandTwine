# BandTwine 可视化编辑器使用指南

可视化编辑器是 BandTwine 互动小说引擎的图形化创作工具，让您通过拖放界面轻松创建复杂的故事分支。

## 🎯 概述

可视化编辑器是一个基于浏览器的 Web 应用，提供：

- **节点图编辑器**：拖放创建故事节点和连接
- **实时属性编辑**：编辑文本、链接、变量等属性
- **完整数据管理**：变量系统、条件逻辑、随机选项、动作系统
- **导入导出**：兼容 BandTwine 原生 JSON 格式
- **主题切换**：深色/浅色模式，护眼舒适

## 🚀 快速开始

### 1. 启动编辑器

```bash
# 进入可视化编辑器目录
cd visual-editor

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

访问 http://localhost:5173（端口可能变化）

### 2. 界面概览

编辑器分为四个主要区域：

```
┌─────────────────────────────────────────────────────┐
│ 顶部导航栏                                          │
├──────────┬───────────────────────────┬──────────────┤
│ 左侧     │     中央画布              │  右侧属性面板 │
│ 工具栏   │    (节点图编辑器)         │              │
├──────────┴───────────────────────────┴──────────────┤
│ 底部状态栏                                          │
└─────────────────────────────────────────────────────┘
```

## 🎮 核心功能详解

### 1. 节点管理

#### 创建节点
1. 点击左侧工具栏的 **"添加节点"** 按钮
2. 新节点会出现在画布中央
3. 拖拽节点调整位置

#### 编辑节点
1. 点击节点选中它
2. 右侧属性面板显示节点属性
3. 编辑文本内容（支持实时预览）

#### 创建连接
1. 选中源节点
2. 在属性面板的 **"链接"** 部分点击 **"添加链接"**
3. 选择目标节点，设置链接文本

### 2. 变量系统

变量是存储游戏状态的容器。编辑器支持三种类型的变量：

#### 变量类型
- **普通变量**：持久化存储的游戏数据
- **显示变量 (show)**：在游戏侧边栏显示的变量（如生命值、分数）
- **临时变量 (temp)**：不保存到存档的临时数据

#### 创建变量
1. 切换到 **"变量管理"** 标签页
2. 在 **"添加新变量"** 部分填写：
   - **路径**：如 `player.health` 或 `show.score`
   - **初始值**：变量的初始值
   - **描述**：（可选）变量用途说明

#### 使用变量
在节点文本中引用变量：
```plaintext
你的生命值：{var.player.health}
当前分数：{var.show.score}
```

### 3. 高级功能

#### 条件系统
创建根据变量值变化的文本选项：

1. 在节点编辑器中找到 **"条件系统"** 部分
2. 点击 **"添加条件集"** 
3. 为每个条件集添加选项，可以设置条件表达式：
   ```javascript
   var.player.level.value > 5  // 等级大于5时显示此选项
   ```

#### 随机系统
创建随机选择的文本内容：

1. 在 **"随机系统"** 部分点击 **"添加随机集"**
2. 添加多个随机选项
3. 可以设置权重和条件控制出现概率

#### 动作系统
定义玩家选择后执行的操作：

##### 节点级动作
- 进入节点时自动执行
- 支持的动作类型：
  - `set`：设置变量值
  - `add`：增加/减少变量值
  - `toggle`：切换布尔值
  - `jump`：跳转到指定节点
  - `toast`：显示消息提示
  - `vibrate`：设备振动
  - `advanceTime`：推进游戏时间
  - `autosave`：自动存档

##### 链接级动作
- 点击链接时执行
- 可以在动作上附加条件

### 4. 图片系统
定义游戏中使用的图片资源：

1. 在 **"图片系统"** 部分点击 **"添加图片定义"**
2. 设置图片 ID、路径和宽度
3. 在文本中使用 `{img.id}` 引用图片

## 📁 导入导出

### 导出故事数据
1. 点击顶部导航栏的 **"文件"** 菜单
2. 选择 **"导出为 JSON"** 
3. 文件将自动下载为 `story-data.json`

### 导入现有故事
1. 点击 **"文件"** → **"导入文件"**
2. 选择 BandTwine JSON 文件
3. 编辑器会自动加载所有节点、变量和数据

### 项目文件
编辑器支持保存完整项目状态：
- **项目文件 (.bandtwine)**：包含节点位置、视口状态等编辑器数据
- **故事文件 (.json)**：仅包含故事数据，兼容 BandTwine 引擎

## 🔧 实用技巧

### 1. 快捷键
- `Ctrl+S` / `Cmd+S`：快速保存
- `Ctrl+Z` / `Cmd+Z`：撤销操作
- `Ctrl+Y` / `Cmd+Y`：重做操作
- `Delete`：删除选中节点（谨慎使用）

### 2. 批量操作
- **多选节点**：按住 `Shift` 键点击多个节点
- **批量编辑**：选中多个节点后，在属性面板编辑通用属性

### 3. 工作流建议
1. **规划故事结构**：先用纸笔规划主要分支
2. **创建核心节点**：建立关键情节节点
3. **添加变量系统**：定义游戏状态变量
4. **实现条件逻辑**：添加选择和分支
5. **测试导出数据**：导出并测试编译

## 🧪 测试与验证

### 1. 测试导出数据
导出的 JSON 数据应该能够被原版 `scripts/compile.js` 编译：

```bash
# 将导出的文件复制到 bandtwine_src 目录
cp story-data.json bandtwine_src/

# 运行编译脚本
node scripts/compile.js
```

### 2. 常见问题排查

#### 编译失败
- 检查变量路径是否正确
- 确保所有链接的目标节点都存在
- 验证 JSON 格式是否正确

#### 编辑器显示问题
- 检查浏览器控制台是否有错误
- 尝试刷新页面或清除缓存
- 确保端口未被占用

## 🚨 注意事项

1. **节点 ID 唯一性**：确保每个节点有唯一的 ID
2. **变量嵌套**：编辑器支持深度嵌套的变量结构
3. **性能优化**：单个故事建议不超过 200 个节点
4. **浏览器兼容**：推荐使用 Chrome、Edge 或 Firefox 最新版
5. **数据备份**：定期导出项目文件备份
